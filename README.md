# Звіт до Практичного Завдання: Unit-тестування програм

## Мета роботи
Вивчення принципів Unit-тестування програмного забезпечення, використання інструментів `pytest` та `pytest-cov`, а також досягнення показника покриття коду тестами не менше 85% для ООП-модуля.

## Хід роботи

### Завдання 1.1: Створення ООП-модуля та Unit-тестів

Було підготовлено необхідну структуру проекту, включно з директоріями `scripts/` та `screens/`. Створено клас **`InventoryManager`** (`scripts/inventory.py`), який містить чотири публічні методи (`add_item`, `get_item`, `update_price`, `remove_item`), що відповідає вимогам ООП.

У файлі `scripts/test_inventory.py` було розроблено 10 unit-тестів, що покривають кожен метод класу (з виконанням вимоги "не менше 2 тестів на функцію"), включаючи перевірку обробки винятків (`ValueError`). Для запуску середовища було встановлено залежності `pytest` та `pytest-cov`.

### Завдання 1.2: Перевірка покриття коду та генерація HTML-звіту

Для коректного збору даних про покриття було здійснено фінальні корекції (додавання `__init__.py`). Була виконана фінальна команда для тестування та генерації звіту: `pytest --cov=scripts.inventory --cov-report html`.

У результаті, всі 10 тестів успішно пройшли, а показник покриття коду модуля `inventory.py` досягнув **100%**, що підтверджено консольним виводом.

## Висновки
Під час виконання практичного завдання було реалізовано повноцінний цикл Unit-тестування. Ми ознайомилися з використанням фреймворку `pytest` для створення ізольованих тестів, а також `pytest-cov` для вимірювання покриття. Досягнуто **100% покриття** коду тестами, що повністю підтверджено згенерованим HTML-звітом.
